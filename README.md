## 🎯 Суть продукта

AutoAssistantAi — это сервис **предиктивного обслуживания автомобилей**, который:

1. **Знает типичные болячки** вашего автомобиля (по марке, модели, поколению, двигателю, КПП)
2. **Предупреждает заранее** — показывает какие проблемы актуальны на вашем пробеге
3. **Даёт решения** — что делать, какие запчасти купить, можно ли сделать самому
4. **Защищает ваши права** — информирует об отзывных кампаниях и коллективных исках

### Ключевая эмоция
> «Я под защитой, всё под контролем»

Пользователь не должен бояться своего автомобиля. Сервис превращает неизвестность в понятный план действий.

## 🧠 Ключевые концепции

### 1. Болячка (Issue)
Типичная проблема конкретного автомобиля. Не абстрактная поломка, а задокументированный опыт тысяч владельцев.

**Структура болячки:**
- Контекст: какой автомобиль затронут (марка, модель, поколение, двигатель)
- Симптомы: что видит/слышит/чувствует владелец
- Причина: почему это происходит
- Пробег: когда проявляется (диапазон km)
- Решение: что делать (сервис / DIY)
- Запчасти: артикулы, цены, где купить
- Статус дефекта: отзывные кампании, коллективные иски

### 2. Индекс здоровья (Health Index)
Число от 0 до 100, показывающее "состояние" автомобиля относительно известных болячек.

**Формула:**
- Базовый индекс: 100
- Минус: сумма весов активных болячек (в зоне риска по пробегу)
- Вес болячки зависит от severity × probability

### 3. Группировка по пробегу
Болячки показываются относительно текущего пробега пользователя:

| Группа | Логика |
|--------|--------|
| **Актуально сейчас** | Пробег внутри диапазона болячки |
| **Скоро** | До начала диапазона < 15,000 км |
| **Пройдено** | Пробег выше конца диапазона |
| **В будущем** | Далеко до начала диапазона |

### 4. Severity (критичность)
| Уровень | Цвет | Описание |
|---------|------|----------|
| Critical | 🔴 Красный | Ехать нельзя, срочный ремонт |
| High | 🟠 Оранжевый | Важно, но можно планировать |
| Medium | 🟡 Жёлтый | Следить за симптомами |
| Low | 🟢 Зелёный | Расходник, не критично |

### 5. Статус дефекта (Defect Status)
Для массовых проблем показываем юридический статус:
- **Коллективные иски** — в каких странах, результаты (выигран/проигран/соглашение)
- **Отзывные кампании** — в каких странах проводились
- **Персонализация по VIN** — применимо ли к конкретному автомобилю

## 🎨 Дизайн-система

### Цвета
```css
--primary: #1F4FD8;        /* Основной синий */

--text-secondary: #64748B; /* Вторичный текст */
--text-tertiary: #94A3B8;  /* Третичный текст */
```

### Принципы UI
1. **Спокойный тон** — не пугаем пользователя, а информируем
2. **Прогрессивное раскрытие** — сначала минимум, детали по запросу
3. **iOS-эстетика** — если выглядит как часть iOS — подходит бренду
4. **Без давления** — "Можно планировать", а не "Срочно чините!"

### Запрещено
- Алармистские формулировки типа "НЕЛЬЗЯ ЕЗДИТЬ!!!"
- Кислотные цвета
- Агрессивные эффекты
- Перегруз информацией

## 🔧 Технический стек

### Frontend
- **React** (Vite)
- **CSS-in-JS** (inline styles)

### Данные
- **JSON** — схема болячек (Gold Standard)

- Deep Research prompts — для сбора болячек
