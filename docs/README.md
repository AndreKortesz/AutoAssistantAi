# AutoAssistantAi — документация

> Эта папка — полное описание сервиса. Любой разработчик, ИИ или LLM должны,
> прочитав эти файлы, понять продукт, архитектуру, все фишки и нюансы — и сразу
> начать работать в контексте проекта.

## С чего начать (порядок чтения)

1. **[`/CLAUDE.md`](../CLAUDE.md)** — корневой контекст: что за продукт, главный принцип
   (спокойный тон), текущее состояние, договорённости и чего нельзя делать.
   Читается автоматически в каждой сессии Claude Code. **Начни отсюда.**
2. **[ARCHITECTURE.md](ARCHITECTURE.md)** — стек, структура файлов, потоки данных, хранилище, роутинг. **Главная техническая карта.**
3. **[HEALTH_INDEX.md](HEALTH_INDEX.md)** — «созревающий» индекс здоровья (модель Oura): уровни, «% картины», веса, классификация болячек по пробегу, дожим. Самая нетривиальная механика — читать обязательно.
4. **[FEATURES.md](FEATURES.md)** — каждая фича по отдельности: онбординг, опрос, болячки, ТО, карта, журнал, ассистент, стоимость владения, коачмарки.
5. **[DATA_MODEL.md](DATA_MODEL.md)** — Gold Standard схема болячки, собранные модели, различия схем RB/HC, как читать поля.

## Справочные документы (валидны, обновляются реже)

- **[CONTRIBUTING.md](CONTRIBUTING.md)** — как запустить, ветки, деплой, code style.
- **[TONE_OF_VOICE.md](TONE_OF_VOICE.md)** — голос бренда, формулировки, антипримеры.
- **[UI_GUIDELINES.md](UI_GUIDELINES.md)** — дизайн-система, токены, компоненты.
- **[../data/ISSUE_SCHEMA.md](../data/ISSUE_SCHEMA.md)** — каноническая схема болячки (v1.1).
- **[PROCESS.md](PROCESS.md)** / **[DATA_COLLECTION.md](DATA_COLLECTION.md)** — процесс сбора данных v4.0 (8 промптов). Это НЕ задача Claude Code, но контекст полезен.

## Принцип поддержания актуальности

Документы ARCHITECTURE / HEALTH_INDEX / FEATURES / DATA_MODEL описывают **код, как он есть сейчас**.
Меняешь поведение — обнови соответствующий раздел в той же правке. Если документ
противоречит коду — верен код, а документ нужно поправить.
