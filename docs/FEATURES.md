# Фичи — экран за экраном

> Что есть в приложении и как оно работает. Индекс здоровья вынесен в
> [HEALTH_INDEX.md](HEALTH_INDEX.md), данные — в [DATA_MODEL.md](DATA_MODEL.md).

## Сквозной путь нового пользователя

```
Онбординг (4 слайда-интро)
   → /add-car (каскадные селекты конфигурации)
   → /checkup (3 вопроса-ощущения + «ещё 2»; «Не знаю» → 3 пути)
   → /dashboard (предварительный индекс, «% картины»)
   → /issues при первом заходе: коачмарк-тур по 3 вкладкам
   → отметка болячек («Уже устранил» / «Ещё не делал» / «Не знаю») → индекс созревает
```
Ничего не обязательно: «Позже»/«Пропустить» есть на каждом шаге, функционал не блокируется.

---

## Онбординг — `Onboarding.jsx`
4 слайда-интро (ценность продукта), знак из `branding/logo-mark-light.png`. По «Начать» —
`markOnboardingCompleted()` + переход на `/add-car`. Показывается только новому пользователю
(флаг в localStorage).

## Добавление авто — `AddCarForm.jsx`
Каскадные селекты как на auto.ru: марка → модель → поколение → двигатель → коробка → год →
пробег → цвет/кузов. Коробки авто-фильтруются по `engine_transmission_compat`. Кузов
спрашивается только если у модели их несколько. По сохранению → `/checkup`.

## Опрос-ощущения — `OnboardingQuestions.jsx` (`/checkup`)
Карточки в стиле Duolingo, но в наших токенах. 3 ядра (двигатель: холодный пуск, расход
масла, звуки) → развилка «Ответить ещё (2)» → разгон + **вопрос по типу коробки** (текст
подставляется по `transmission.type`: АКПП / Вариатор / Робот / МКПП). На каждом вопросе —
«Не знаю / только купил» (пишет `unknown`, не штрафует) → пути: проверить самому / спросить в
сервисе / **спросить ассистента (отложенно)**. «Спросить ассистента» не уводит из опроса —
сохраняет вопрос в `deferredQuestions` (localStorage); на главной появляется мягкий нудж
«Вы хотели уточнить … — спросить ассистента?» → `/assistant` с предзаполненным вопросом
(через `location.state.prompt`). Опрос **возобновляется** (стартует с первого неотвеченного/
«не знаю»), поэтому «напомнить позже» реально возвращает вопрос. «Позже» сверху — с
подтверждением. Цвет только в иконке-маркере ответа. Каждый ответ сразу сохраняется
(`saveAnswers`) → индекс пересчитывается.

## Главная — `Dashboard.jsx` («Мой гараж»)
- Шапка: знак + «Мой гараж».
- Гео-карточка: силуэт авто (`CarSilhouette` по типу/цвету), название, двигатель, пробег
  (тап → `MileageUpdateModal`).
- **Кольцо зрелости индекса** + статус + «% картины» + адаптивный CTA (см. HEALTH_INDEX.md §5).
- Полоса систем (двигатель/подвеска/тормоза/КПП), разворачивается.
- **«Стоимость владения в год»** (сворачивается; подробно — `/cost`).
- Чип «Спросить про мою машину» → `/assistant` (с примером вопроса).
- Топ-приоритет: самая тяжёлая актуальная неотмеченная болячка + «Я починил».
- Анимация «+N» при отметке «сделано» (тихая).

## Обслуживание — `IssuesScreen.jsx` (`/issues`), 3 вкладки

### Слабые места
Болячки по важности: `safety` / `planned` / `minor` / `upcoming` («впереди») и отдельная
свёрнутая куча **«Сделано»**. Карточки `IssueCard` — раскрываемые, с иконками иска/отзывной
(`scale`/`fileText`), быстрыми метриками и **дожимом** (см. HEALTH_INDEX.md §6). Отмеченные
«устранено» уходят из активных групп в «Сделано» (оттуда — «Вернуть»). Сверху — баннер
«Отзывные кампании и иски» по поколению; 3 чипа «сейчас / впереди / пройдено». «Сейчас» в
заголовке не считает мелочи (не пугаем цифрой). Усечение групп: первые 3 + «Ещё N».

### ТО и расходники — `MaintenanceTab.jsx`
Две подсекции: **Регламент** (по графику, иконка calendar) и **Износ** (по состоянию/пробегу,
иконка gauge). Строки кликабельны (стрелка) → открывают детальную карточку (что менять,
причина, интервал, симптомы, артикулы, РФ-нюансы). По умолчанию показывается актуальное;
«Показать всё, что впереди (N)» раскрывает раннее.

### Карта — `MileageMapTab.jsx`
Вертикальный таймлайн «дорога вперёд» по пробегу: болячки (тапаемые), ТО, износ + «Вы здесь».
Пройденное скрыто за «Показать пройденное (N)».

## Детальная болячка — `IssueDetailScreen.jsx` (`/issues/:id`)
Открывает любую запись по id (болячка / ТО / износ). Контекст → симптом → причина → пробег →
решение (diy/сервис) → запчасти (артикулы, ревизии, аналоги, «не брать») → юр-статус (иски,
recalls, TSB с флагами стран) → owner_reports → источники. Кнопка «Уже устранил» с превью
«+N» к индексу (считается **с учётом ответов** — синхронно с дашбордом).
Поля бывают строкой вместо массива → защищены `asArray()`.

## Журнал — `JournalScreen.jsx`
История ТО/ремонтов (`journalService`). Отметки «устранено» болячек тоже попадают сюда; из
журнала выводятся `fixedIssueIds` (с TTL по пробегу — со временем отметка «стареет»).

## Ассистент — `AssistantScreen.jsx` (`/assistant`)
AI-чат через `POST /api/chat` → Gemini 2.5 Flash (ключ только на сервере). Системный промпт
заземлён на болячки конфигурации, запрет выдумывать артикулы/recall/цены. Ответы рендерятся
как **markdown** (жирный, списки, абзацы) — не сырой текст. Быстрые подсказки-вопросы.

## Стоимость владения — `CostScreen.jsx` (`/cost`)
Полная годовая стоимость владения (не только сервис): топливо, ТО/расходники, страховка,
налог и т.д. — оценка из допущений (`estimateOwnership`, `OWNERSHIP_ASSUMPTIONS`), допущения
показываются честно. На главной — свёрнутая сводка.

## Коачмарк-тур — `CoachmarksTour.jsx`
При первом заходе в «Обслуживание» (с реальными данными). Затемняет экран, «вырезает»
подсветку на вкладке (box-shadow), баблик с подписью + «Далее»/«Пропустить». 3 шага по
вкладкам на данных конкретной машины. Не слайды. «Просмотрено» — в localStorage.

## Сервисы — `ServicesScreen.jsx` (`/services`, 5-й пункт меню)
Витрина услуг (стиль Т-Банка, но в нашем светлом стиле). Порядок сверху вниз: промо-баннер
→ «Помощь в дороге» (раскрывается в список: эвакуатор, прикурить, вскрытие, топливо, ДТП) →
«Выбираете машину?» (чек-лист б/у, проверка истории, выездной эксперт, объявления) → «Уже
есть машина» (сетка 2 кол.: ОСАГО/КАСКО, автосервисы, запчасти, шиномонтаж, юрпомощь, «Все
сервисы») → плашка про партнёров. **Все сервисы — «В разработке».** Тап по сервису → отдельная страница
`ServiceDetailScreen` (`/services/:id`, каталог в `servicesCatalog.js`): описание + бейдж
«В разработке» + кнопка **«Уведомить о запуске»**. Нажатие = `trackServiceInterest(serviceId)`
(`serviceAnalytics`): POST `/api/interest` (сервер логирует + шлёт владельцу в Telegram/вебхук,
если заданы env) + локальный счётчик + состояние «Уведомим ✓». Счётчики пользователю не видны
(внутренняя аналитика владельца). Иконки сервисов — один info-цвет, без заливок. «Чек-лист подбора б/у» — реальная фича (помечен «готово»), ведёт
на `/checklist` (см. ниже).

## Чек-лист покупки — `BuyerChecklistScreen.jsx` (`/checklist`)
Что проверить перед покупкой именно этой конфигурации. **Собирается динамически из болячек**
(`issuesData.systemic`, critical/high/medium), а не из статичного текста: каждый пункт — чекбокс
«проверил» + заголовок (точка severity) + «как проверить» (из `diagnostic.visual_check` →
`instruction` → симптом) + диапазон стоимости ремонта; тап по пункту → детальная болячка.
Сверху — блок «Сначала пробей VIN» (учитывает число отзывных по поколению), прогресс «N из M».
Отметки эфемерные (осмотр чужой машины). Спокойный тон: «места внимания, а не приговор».

## Иконки и силуэт
`Icon.jsx` — инлайн-SVG (Lucide-стиль), **только контурные**, без эмодзи (владелец это любит).
`CarSilhouette.jsx` — SVG-силуэт по типу кузова и цвету из палитры.
